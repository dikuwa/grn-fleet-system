# Handover Guide

## Deliverables

- Git repository and protected main branch
- Architecture and schema documentation
- Environment variable inventory
- Deployment access list
- Database migration and backup procedures
- R2 bucket and retention configuration
- Email/maps/job/monitoring setup
- Tenant onboarding guide
- User/role guide
- Import templates
- Admin and driver user guides
- Test report and known limitations
- Incident and support contacts

## Owner handover session

Demonstrate:

1. Create tenant/office.
2. Import staff and vehicles.
3. Activate a user and assign acting role.
4. Submit regional and national requests.
5. Approve, allocate, release and authorise.
6. Complete inspections and issue.
7. Driver offline log and fuel record.
8. Return, defect and closure.
9. Generate/share/revoke document.
10. Run reports and audit export.
11. Suspend account and rotate credentials.

## Credential rules

Transfer through a secure password manager. Rotate temporary implementation credentials. Record owners and recovery methods. Never put production secrets in the handover PDF or repository.

## Warranty/support readiness

Maintain a list of current version, deployment date, database migration, known issue, monitoring link and rollback point. Support access must be time-limited and audited.
